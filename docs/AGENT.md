# AGENT.md (Folder-Level Guide)

## Purpose of this folder

This folder (`docs/`) contains documentation including architecture, onboarding, development guides, and runbooks.

## What agents may do here

- Update documentation when code behavior changes (Principle 19: Docs Age With Code)
- Add new documentation for new features
- Update examples when code changes (Principle 20: Examples Are Contracts)
- Improve documentation clarity and completeness

## What agents may NOT do

- Leave documentation outdated when code changes
- Create documentation that contradicts code
- Break documentation structure without justification
- Remove documentation without replacement
- Create examples that don't match actual code behavior

## Required links

- Refer to higher-level policy: `.repo/policy/PRINCIPLES.md` (Principle 19: Docs Age With Code, Principle 20: Examples Are Contracts)
- See `.repo/policy/BESTPR.md` for documentation expectations
- See `docs/ARCHITECTURE.md` for system design
- See `docs/DEVELOPMENT.md` for development workflow

## Documentation Standards

- Documentation must match code behavior
- Examples must be runnable and correct
- Architecture docs must reflect actual structure
- Onboarding docs must be kept current
- All docs should include filepaths where relevant
